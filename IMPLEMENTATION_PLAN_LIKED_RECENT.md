Критерий готовности:

## 3. SpotifyClient расширение

- [ ] Добавить API-метод чтения liked tracks с пагинацией.
- [ ] Возвращать URI в порядке newest -> oldest.
- [ ] Игнорировать невалидные элементы без URI.

Критерий готовности:
- Метод стабильно возвращает последние `N` liked URI.

## 4. Менеджер плейлистов для окон

- [ ] Реализовать построение имени плейлиста по формуле.
- [ ] Реализовать `find-or-create` для каждого окна.
- [ ] Переиспользовать существующие методы replace items.

Критерий готовности:
- Для всех окон из env существуют корректные целевые плейлисты.

## 5. Scheduler синхронизации

- [ ] Реализовать периодический процесс sync (таймер).
- [ ] На каждом цикле загружать `maxWindow`.
- [ ] Формировать окна `N` через `slice(0, N)`.
- [ ] Выполнять replace для каждого окна.
- [ ] Добавить `syncNow()` для ручного/стартового запуска.
- [ ] Добавить graceful stop.

Критерий готовности:
- Синхронизация работает циклически и не падает при остановке приложения.

## 6. Оптимизация лишних обновлений

- [ ] Добавить in-memory hash/snapshot по каждому окну.
- [ ] Пропускать `PUT`, если окно не изменилось.
- [ ] Опционально сохранять кэш в локальный state-файл.

Критерий готовности:
- При неизменных liked-треках лишние запросы `PUT` не выполняются.

## 7. Ошибки и ретраи

- [ ] Обработать `401` через refresh + retry.
- [ ] Обработать `429` с учетом `Retry-After`.
- [ ] При сетевых ошибках логировать `WARN` и продолжать следующий цикл.
- [ ] Изолировать ошибки окон, чтобы сбой одного окна не ломал остальные.

Критерий готовности:
- Сервис устойчив к временным проблемам API.

## 9. Ручная приемка

- [ ] Запустить с `LIKED_RECENT_WINDOWS=20,50,100`.
- [ ] Добавить трек в избранное, дождаться обновления.
- [ ] Удалить трек из избранного, дождаться обновления.
- [ ] Проверить корректные размеры окон.
- [ ] Проверить, что порядок всегда newest -> oldest.
- [ ] Проверить, что плейлисты обновляются без дублей лишних запросов (по логам).

Критерий готовности:
- Фича полностью работает на реальном аккаунте Spotify.
