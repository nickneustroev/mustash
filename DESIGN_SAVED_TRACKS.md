# ТЗ: Saved Tracks Sync (Избранные треки)

## 1. Цель

Создать сервис синхронизации, который сохраняет в локальной базе данных полный список избранных треков Spotify и поддерживает его в актуальном состоянии.

## 2. Требования к данным

Хранить для каждого трека:
- `trackId` - уникальный ID трека (из Spotify URI)
- `trackUri` - полный Spotify URI
- `trackName` - название трека
- `artistName` - имя основного исполнителя
- `addedAt` - дата и время добавления в избранное

## 3. Ожидаемое поведение

1. При запуске сервис выполняет полную синхронизацию (fetch all tracks).
2. По таймеру сервис синхронизирует изменения (новые лайки, удаленные лайки).
3. Локальная БД всегда отражает актуальное состояние избранного.
4. Данные в БД обновляются инкрементально (не перезаписываются полностью).

## 4. Конфигурация

Новые env-параметры:

| Параметр | Тип | По умолчанию | Описание |
|----------|-----|--------------|----------|
| `SAVED_TRACKS_ENABLED` | boolean | false | Включить синхронизацию |
| `SAVED_TRACKS_SYNC_INTERVAL_MS` | number | 60000 | Интервал синхронизации (мин 15000) |

## 5. База данных

### Schema

```prisma
model SavedTrack {
  trackId    String   @id
  trackUri   String   @unique
  trackName  String?
  artistName String?
  addedAt    DateTime
  updatedAt  DateTime @updatedAt

  @@index([trackId])
}
```

## 6. Архитектура

### 6.1 Расширение SpotifyClient

Новые методы:

1. `getSavedTracksFull(limit: number, offset: number)` - возвращает полную информацию о треках с пагинацией
2. `getSavedTracksCount()` - возвращает общее количество избранных треков

### 6.2 SavedTrackRepository

Интерфейс:
- `upsertSavedTrack(track: SavedTrackData)` - вставка или обновление
- `deleteSavedTrack(trackId: string)` - удаление
- `getAllSavedTrackIds()` - получить все ID
- `getSavedTrackCount()` - количество записей
- `getAllSavedTracks()` - все треки

### 6.3 SavedTracksSyncService

API:
- `start()` - запустить синхронизацию
- `stop()` - остановить
- `syncNow()` - принудительная синхронизация

Алгоритм синхронизации:

```
1. Получить текущее состояние из Spotify (все треки)
2. Получить текущее состояние из БД
3. Найти новые треки (есть в Spotify, нет в БД) -> upsert
4. Найти удаленные треки (есть в БД, нет в Spotify) -> delete
5. Обновить метаданные для существующих (если изменились)
```

## 7. Обработка ошибок

- 401 -> refresh token + retry
- 429 -> лог + ожидание до следующего цикла
- Сетевые ошибки -> warning, пропуск цикла
- Ошибки БД -> лог, остановка синхронизации

## 8. Логирование

События:
1. Старт/стоп сервиса
2. Количество треков в избранном (при старте)
3. Результаты синхронизации: new=X, updated=Y, removed=Z
4. Ошибки API и БД

## 9. Критерии приемки

1. После запуска все избранные треки сохраняются в БД с полными метаданными.
2. Новый лайк автоматически добавляется в БД.
3. Снятый лайк автоматически удаляется из БД.
4. Измененные метаданные (название, исполнитель) обновляются в БД.
5. Сервис стабильно работает при перезапусках.
6. Функционал включается/выключается через env-параметр.
